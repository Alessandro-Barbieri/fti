41 /* In all of the routines below:
42 
43  k = Number of data devices
44  m = Number of coding devices
45  w = Word size
46 
47  data_ptrs = An array of k pointers to data which is size bytes.
48  Size must be a multiple of sizeof(long).
49  Pointers must also be longword aligned.
50 
51  coding_ptrs = An array of m pointers to coding data which is size bytes.
52 
53  packetsize = The size of a coding block with bitmatrix coding.
54  When you code with a bitmatrix, you will use w packets
55  of size packetsize.
56 
57  matrix = an array of k*m integers.
58  It represents an m by k matrix.
59  Element i,j is in matrix[i*k+j];
60 
61  bitmatrix = an array of k*m*w*w integers.
62  It represents an mw by kw matrix.
63  Element i,j is in matrix[i*k*w+j];
64 
65  erasures = an array of id's of erased devices.
66  Id's are integers between 0 and k+m-1.
67  Id's 0 to k-1 are id's of data devices.
68  Id's k to k+m-1 are id's of coding devices:
69  Coding device id = id-k.
70  If there are e erasures, erasures[e] = -1.
71 
72  schedule = an array of schedule operations.
73 
74  If there are m operations, then schedule[m][0] = -1.
75 
76  operation = an array of 5 integers:
77 
78  0 = operation: 0 for copy, 1 for xor (-1 for end)
79  1 = source device (0 - k+m-1)
80  2 = source packet (0 - w-1)
81  3 = destination device (0 - k+m-1)
82  4 = destination packet (0 - w-1)
83  */

87 /*
88  - jerasure_matrix_to_bitmatrix turns a m X k matrix in GF(2^w) into a
89  wm X wk bitmatrix (in GF(2)). This is
90  explained in the Cauchy Reed-Solomon coding
91  paper.
92 
93  - jerasure_dumb_bitmatrix_to_schedule turns a bitmatrix into a schedule
94  using the straightforward algorithm -- just
95  schedule the dot products defined by each
96  row of the matrix.
97 
98  - jerasure_smart_bitmatrix_to_schedule turns a bitmatrix into a schedule,
99  but tries to use previous dot products to
100  calculate new ones. This is the optimization
101  explained in the original Liberation code paper.
102 
103  - jerasure_generate_schedule_cache precalcalculate all the schedule for the
104  given distribution bitmatrix. M must equal 2.
105 
106  - jerasure_free_schedule frees a schedule that was allocated with
107  jerasure_XXX_bitmatrix_to_schedule.
108 
109  - jerasure_free_schedule_cache frees a schedule cache that was created with
110  jerasure_generate_schedule_cache.
111  */

140 /* These return integers, because the matrix may not be invertible.
141 
142  The parameter row_k_ones should be set to 1 if row k of the matrix
143  (or rows kw to (k+1)w+1) of th distribution matrix are all ones
144  (or all identity matrices). Then you can improve the performance
145  of decoding when there is more than one failure, and the parity
146  device didn't fail. You do it by decoding all but one of the data
147  devices, and then decoding the last data device from the data devices
148  and the parity device.
149 
150  jerasure_schedule_decode_lazy generates the schedule on the fly.
151 
152  jerasure_matrix_decode only works when w = 8|16|32.
153 
154  jerasure_make_decoding_matrix/bitmatrix make the k*k decoding matrix
155  (or wk*wk bitmatrix) by taking the rows corresponding to k
156  non-erased devices of the distribution matrix, and then
157  inverting that matrix.
158 
159  You should already have allocated the decoding matrix and
160  dm_ids, which is a vector of k integers. These will be
161  filled in appropriately. dm_ids[i] is the id of element
162  i of the survivors vector. I.e. row i of the decoding matrix
163  times dm_ids equals data drive i.
164 
165  Both of these routines take "erased" instead of "erasures".
166  Erased is a vector with k+m elements, which has 0 or 1 for
167  each device's id, according to whether the device is erased.
168 
169  jerasure_erasures_to_erased allocates and returns erased from erasures.

171  */

221 /* ------------------------------------------------------------ */
222 /* Matrix Inversion ------------------------------------------- */
223 /*
224  The two matrix inversion functions work on rows*rows matrices of
225  ints. If a bitmatrix, then each int will just be zero or one.
226  Otherwise, they will be elements of gf(2^w). Obviously, you can
227  do bit matrices with crs_invert_matrix() and set w = 1, but
228  crs_invert_bitmatrix will be more efficient.
229 
230  The two invertible functions return whether a matrix is invertible.
231  They are more efficient than the inverstion functions.
232 
233  Mat will be destroyed when the matrix inversion or invertible
234  testing is done. Sorry.
235 
236  Inv must be allocated by the caller.
237 
238  The two invert_matrix functions return 0 on success, and -1 if the
239  matrix is uninvertible.
240 
241  The two invertible function simply return whether the matrix is
242  invertible. (0 or 1). Mat will be destroyed.
243  */
244
